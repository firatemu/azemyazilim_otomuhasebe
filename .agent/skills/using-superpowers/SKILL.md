<EXTREMELY-IMPORTANT>
If a skill applies to your task, YOU DO NOT HAVE A CHOICE. YOU MUST USE IT.
Invoke relevant or requested skills BEFORE any response or action. Even a 1% chance a skill might apply means you MUST invoke it.
</EXTREMELY-IMPORTANT>

## Interaction Pattern
1. **User message received**.
2. **Might any skill apply?** (Yes, even 1%).
3. **Invoke Skill tool**.
4. **Announce**: "Using [skill] to [purpose]".
5. **Has checklist?** -> Create TodoWrite.
6. **Follow skill exactly**.

## Red Flags (Rationalizations to Avoid)
- "This is just a simple question".
- "I need more context first".
- "I'll just do this one thing first".
- "The skill is overkill".
- "I know what that means (knowing != using)".

## Skill Priority
1. **Process skills first** (e.g., brainstorming, debugging).
2. **Implementation skills second** (e.g., mcp-builder, frontend-design).
